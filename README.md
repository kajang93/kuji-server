## 📈 단계별 개발 로드맵 (Milestones)

현재 인프라와 보안의 기틀, 그리고 가장 까다로운 OAuth 2.0 회원 연동이 완료되었습니다.
이제 부가 기능(프로필, 배송지) 개발은 뒤로 미루고, 서비스의 핵심 비즈니스 로직인 **'쿠지 엔진(랜덤 뽑기)'** MVP(최소 기능 제품) 개발에 모든 리소스를 집중합니다.

### ✅ Phase 1: 인프라 및 보안/인증 코어 (완료)
- [x] DB 설계 및 PostgreSQL 13개 테이블 수동 생성 및 매핑 (`ddl-auto: validate`)
- [x] Spring Security + BCrypt 암호화 및 JWT Stateless 인증 필터 구현
- [x] `@RestControllerAdvice` 전역 예외 처리 및 공통 응답 규격화
- [x] 엔티티 공통 필드 매핑 (`BaseTimeEntity`)

### 🚧 Phase 2: 쿠지(Kuji) 핵심 엔진 빌드 (Active 🎯)

#### 2.1 회원 및 인증 통합 (완료)
- [x] 카카오 OAuth 2.0 연동 및 Kuji JWT 토큰 발급 로직 구축
- [x] 프론트엔드-백엔드 인증 통합 및 로컬 스토리지 기반 로그인 유지

#### 2.2 MVP 1단계: 쿠지 판 전시 (현재 집중 구간)
- [ ] **엔티티 설계**: `KujiBoard`(판) 단일 엔티티 뼈대 생성 및 수동 데이터 삽입
- [ ] **리스트 API**: 메인 화면 전시용 `GET /api/kuji` 리스트 조회 API 구축
- [ ] **프론트 연동**: React 메인 화면에 쿠지 리스트 카드(`KujiCard`) 렌더링

#### 2.3 MVP 2단계: 상품 정보 및 확률 엔진
- [ ] **상품 매핑**: `KujiItem`(상품), `KujiGrade`(등급) 엔티티 설계 및 `KujiBoard`와 연관관계 매핑
- [ ] **상세 API**: 특정 쿠지 판의 상품별 남은 수량을 보여주는 상세 조회 API 구현
- [ ] **뽑기 로직**: 전체 수량/남은 수량 기반 무작위 상품 추출 알고리즘 구현
- [ ] **재고 차감**: 상품 당첨 시 `remain_qty` 차감 및 당첨 이력(`DrawResult`) 저장

#### 2.4 경제 시스템 및 동시성 제어
- [ ] **재고 잠금**: 뽑기 시 발생할 수 있는 데이터 경합을 **비관적 락(Pessimistic Lock)**으로 해결
- [ ] **포인트/결제**: 사용자 포인트 증감 로직 및 포트원(PortOne) 실결제 연동

### 🏗️ Phase 3: 서비스 확장 및 부가 기능 고도화

#### 3.1 마이페이지 및 프로필 (후순위)
- [ ] **사용자 정보**: 가입일, 생년월일 수정 기능
- [ ] **물류 연동**: 당첨 상품 수령을 위한 배송지 입력 및 관리 기능
- [ ] **이미지 스토리지**: AWS S3를 활용한 회원 프로필 사진 및 쿠지 썸네일 등록

#### 3.2 커뮤니티 및 알림
- [ ] **커뮤니티**: 자유 게시판, 인증 샷 게시판 및 댓글/좋아요 기능
- [ ] **푸시 알림**: FCM(Firebase Cloud Messaging) 연동 및 알림 설정

### 🚀 Phase 4: 프로덕션 운영 및 출시
- [ ] **배포 자동화**: Railway CI/CD 파이프라인 구축 및 무중단 배포
- [ ] **하이브리드 앱**: Capacitor 빌드 및 스토어(Android/iOS) 최종 심사
